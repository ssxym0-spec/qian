# 场景化应用卡片样式更新说明

## 📋 更新概述

**更新日期**: 2025年10月13日  
**更新内容**: 将场景化应用从轮播展示改为网格卡片展示  
**影响范围**: `/yunyang` 页面的"场景化应用探索"区块

---

## 🎯 更新前后对比

### 更新前 ❌
- **展示方式**: 左右滑动的轮播
- **一次显示**: 1个场景
- **交互方式**: 拖拽切换、自动播放
- **信息密度**: 低（需要逐个查看）

### 更新后 ✅
- **展示方式**: 网格卡片布局
- **一次显示**: 多个场景（桌面端3个，平板2个，手机1个）
- **交互方式**: 点击卡片查看详情弹窗
- **信息密度**: 高（一屏浏览所有场景）

---

## 🎨 新设计特性

### 1. 卡片展示
- **网格布局**: 响应式3列（桌面）/ 2列（平板）/ 1列（移动）
- **卡片样式**: 白色背景 + 蓝色顶部边框 + 圆角阴影
- **Hover效果**: 上浮 + 阴影加深
- **图标**: 大号Emoji图标（48px）

### 2. 信息结构
每个卡片清晰展示：
- 🎯 **场景图标 + 标题**
- 💭 **痛点**（黄色背景卡片）
- 💡 **方案**（蓝色背景卡片）
- ✨ **核心价值**（绿色标签）
- 📱 **查看详情按钮**（渐变按钮）

### 3. 详情弹窗
点击卡片或"查看详情"按钮后：
- **全屏遮罩**: 黑色半透明 + 背景模糊
- **弹窗内容**: 居中显示，最大宽度800px
- **详细信息**: 完整展示所有场景信息
- **行动按钮**: 立即咨询 + 关闭
- **关闭方式**: 点击遮罩 / 点击关闭按钮

## 🔧 技术实现

### 数据兼容性

组件同时支持**新格式**和**旧格式**数据：

#### 新格式（推荐）
```typescript
{
  id: "1",
  title: "商务礼品",
  icon: "🎁",
  pain_point: "企业送礼千篇一律",
  solution: "认养专属茶园",
  background_image: "/images/bg.jpg",
  core_values: [
    { icon: "💎", title: "高端定制", description: "..." }
  ]
}
```

#### 旧格式（自动解析）
```typescript
{
  id: "1",
  title: "商务礼品",
  content: "痛点：企业送礼千篇一律\n方案：认养专属茶园\n价值：高端定制"
}
```

### 智能解析

如果使用旧的`content`格式，组件会自动：
1. 识别关键词（痛点、方案、价值）
2. 提取结构化信息
3. 转换为新格式展示
4. 自动匹配图标

## 🎨 样式定制

### 修改主题色

卡片顶部边框颜色（蓝色主题）：
```tsx
className="border-t-4 border-[#3498db]"
```

可改为紫色（企业领养主题）：
```tsx
className="border-t-4 border-[#9b59b6]"
```

### 修改按钮样式

查看详情按钮渐变：
```tsx
className="bg-gradient-to-r from-blue-500 to-purple-600"
```

## 🐛 常见问题

### Q1: 卡片显示不全？
**A**: 检查数据格式，确保至少有`title`和`content`或`pain_point`字段。

### Q2: 弹窗无法关闭？
**A**: 检查浏览器控制台是否有JavaScript错误。

### Q3: 响应式布局异常？
**A**: 清除浏览器缓存，确保Tailwind CSS正确加载。

### Q4: 图标不显示？
**A**: 确保使用的是标准Emoji字符，浏览器需支持Emoji显示。
