## 🔧 技术实现

### 1. 组件架构

```
ScenariosPage (页面容器)
├── ScenarioCard (场景卡片) × N
│   ├── 场景图标
│   ├── 场景标题
│   ├── 痛点描述
│   ├── 解决方案
│   ├── 核心价值标签
│   └── 查看详情按钮
└── ScenarioModal (详情弹窗)
    ├── 场景头部
    ├── 背景图片
    ├── 痛点分析
    ├── 解决方案
    ├── 核心价值详情
    └── 行动按钮
```

### 2. 状态管理

使用React Hooks管理组件状态：
- `useState` - 场景列表、选中场景、弹窗状态
- `useEffect` - 数据获取、URL参数监听
- `useSearchParams` - URL参数读取

### 3. 数据流

```
1. 页面加载 → 读取URL参数
2. 调用API → 获取场景数据
3. 渲染卡片 → 展示场景列表
4. 点击卡片 → 打开详情弹窗
5. 查看详情 → 显示完整信息
6. 关闭弹窗 → 返回列表
```

### 4. API集成

```javascript
// 接口地址
GET /api/adoption-plans/{type}

// 参数
type: 'private' | 'enterprise'

// 返回数据
{
  type: string,
  scenario_applications: Scenario[]
}
```

## 📞 常见问题

### Q1: 如何自定义主题色？
**A**: 修改 `scenarios.css` 中的颜色变量：
```css
.scenario-card-private {
  border-top-color: #your-color;
}
```

### Q2: 如何集成客服系统？
**A**: 修改 `ScenarioModal.tsx` 中的 `handleContactUs` 函数：
```tsx
const handleContactUs = () => {
  // 集成你的客服系统
  window.open('your-customer-service-url');
};
```

### Q3: 如何添加新的场景？
**A**: 通过管理后台添加场景数据，前端会自动展示。或者直接调用后端API添加。

### Q4: 数据从哪里来？
**A**: 从后端API获取：`GET /api/adoption-plans/{type}`

### Q5: 支持哪些设备？
**A**: 支持所有现代浏览器和设备，包括桌面端、平板端、移动端。
